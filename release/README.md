# Lexicon Release Build

This directory contains the release build pipeline and Windows installer for Lexicon.

## Prerequisites

- **Node.js 18+** with npm (for frontend build)
- **Go 1.22+** (for backend build)
- **Inno Setup 6+** (for installer compilation)
  - Download from https://jrsoftware.org/isdl.php
  - Default install path: `C:\Program Files (x86)\Inno Setup 6\iscc.exe`

## Build

Run the PowerShell build script from the **repo root** or from this directory:

```powershell
powershell -ExecutionPolicy Bypass -File release/build.ps1
```

The script performs these steps:

1. **Build frontend** — runs `npm run build` in `frontend/`, producing `frontend/dist/`
2. **Copy dist** — copies `frontend/dist/` into `backend/cmd/server/dist/` for Go embedding
3. **Build Go binary** — compiles `lexicon.exe` with embedded static files (`-ldflags "-s -w"`)
4. **Gather tools** — copies `tools/spotiflac.exe` into `release/tools/`
5. **Compile installer** — runs Inno Setup compiler (`iscc`) on `release/lexicon.iss`

**Output:** `release/LexiconSetup.exe`

## Manual Steps (if build.ps1 fails)

```powershell
# 1. Build frontend
cd ..\frontend
npm install
npm run build

# 2. Copy dist to backend embed directory
Remove-Item -Recurse -Force ..\backend\cmd\server\dist -ErrorAction SilentlyContinue
Copy-Item -Recurse ..\frontend\dist ..\backend\cmd\server\dist

# 3. Build Go binary
cd ..\backend
go build -ldflags "-s -w" -o ..\release\lexicon.exe .\cmd\server

# 4. Copy tool binaries
Copy-Item ..\tools\spotiflac.exe ..\release\tools\spotiflac.exe -ErrorAction SilentlyContinue

# 5. Compile installer
cd ..\release
iscc lexicon.iss
```

## Installer Features

The `LexiconSetup.exe` wizard guides the user through:

1. **Installation folder** — standard directory selection
2. **DeepSeek API Key** — password input with format validation (`sk-` prefix)
3. **Media Folders** — folder browser for music library paths (multi-select, semicolon-separated)
4. **Optional Tools** — paths for external binaries:
   - Spotify Client ID / Secret (optional)
   - `spotdl.exe` path (optional)
   - `yt-dlp.exe` path (optional)
   - `ffmpeg.exe` path (optional)
5. **Shortcuts** — optional Desktop shortcut

Post-install, the wizard writes `{app}\.env` with all configured values and creates Start Menu shortcuts.

## Runtime Behavior

- The Go backend (`lexicon.exe`) embeds the built React frontend and serves it on the configured port (default `8787`)
- API routes (`/api/*`) are registered before the SPA catch-all, so they take priority
- CORS is disabled in production (frontend and backend share the same origin)
- `.env` is read from the working directory (set to `{app}` by the shortcuts)
- User data (`data/lexicon.db`) and config (`.env`) are preserved across reinstalls

## Architecture

```
LexiconSetup.exe (Inno Setup)
├── lexicon.exe          ← Go backend + embedded React frontend
├── tools/
│   └── spotiflac.exe    ← bundled download tool
├── data/                ← SQLite database (created at runtime)
└── .env                 ← config file (generated by installer)
```

## Troubleshooting

| Issue | Cause | Fix |
|-------|-------|-----|
| `go:embed` build fails | `backend/cmd/server/dist/` missing | Run frontend build + copy step first |
| `iscc not found` | Inno Setup not in PATH | Use full path or add to PATH |
| Frontend shows placeholder | Old `dist/` cached in embed | Clean and rebuild: `Remove-Item backend/cmd/server/dist -Recurse` |
| Port 8787 in use | Another app listening | Change `PORT` in `.env` and restart |
