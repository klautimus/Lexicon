## A. Issue #13 — Download State Lost on Navigation

### A.1 Design Decision

**Chosen approach: React Context (DownloadContext)**

A new `DownloadContext` wraps the app and holds all download/progress state that currently lives in `RecsPage` component state. The context provider never unmounts during route navigation, so state persists when moving between pages.

**Alternatives considered and rejected:**

| Alternative | Why rejected |
|-------------|-------------|
| Store in PlayerContext | Pollutes PlayerContext with unrelated download state. PlayerContext is for audio playback, not download tracking. |
| URL query-string state | URLs get ugly with many IDs, doesn't work well with React Router for transient state, can't track polling intervals. |
| localStorage | Polling intervals are cleared on unmount — can't resume live polling from storage alone. Adds stale-data risk. |
| Poll backend on page load | Backend jobs are in-memory (lost on restart), doesn't track which discover items were being downloaded, needs new API endpoints. Overengineered. |
| Hybrid (Context + backend polling) | Overengineered. Context alone solves the navigation problem. |

**Why DownloadContext wins:**
- Follows the existing pattern (`ToastContext`, `PlayerContext`)
- Zero backend changes
- State persists across route navigation (just not full page refresh — acceptable, same as ToastContext)
- Clean separation of concerns
- Simple implementation (~100 lines new code in context, ~50 lines removed from RecsPage)

### A.2 Architecture

```
App.tsx
  ToastProvider
    PlayerProvider
      DownloadProvider          ← NEW: wraps entire app
        Router
          RecsPage              ← uses useContext(DownloadContext)
          MusicPage             ← can also access download state
          SearchPage            ← can also access download state
```

**State the context manages** (currently all in RecsPage `useState`):

| State | Type | Purpose |
|-------|------|---------|
| `downloadingIds` | `Set<string>` | Tracks actively downloading items |
| `completedIds` | `Set<string>` | Tracks completed downloads (discover items) |
| `completedTrackIds` | `Record<string, number>` | Maps name → track_id for "Play" button |
| `playlistPreview` | `PlaylistPayload \| null` | AI-generated playlist before creation |
| `playlistTrackStatus` | `Record<string, status>` | Per-track status in playlist creation |
| `createdPlaylistId` | `number \| null` | ID of created playlist (for "View Playlist" link) |
| `generatingPlaylist` | `boolean` | Loading state for playlist generation |
| `creatingPlaylist` | `boolean` | Loading state for playlist creation |

#### Task 13.1: Create DownloadContext

**File:** Create `frontend/src/contexts/DownloadContext.tsx`

**Step 1: Write the context file**

```typescript
import {
  createContext,
  useContext,
  useState,
  useCallback,
  ReactNode,
} from "react";
import { PlaylistPayload } from "../lib/api";

type TrackStatus = "pending" | "present" | "downloading" | "completed" | "failed";

interface DownloadState {
  // Discover-item download tracking
  downloadingIds: Set<string>;
  completedIds: Set<string>;
  completedTrackIds: Record<string, number>;
  
  // AI playlist generation
  playlistPreview: PlaylistPayload | null;
  playlistTrackStatus: Record<string, TrackStatus>;
  createdPlaylistId: number | null;
  generatingPlaylist: boolean;
  creatingPlaylist: boolean;
}

interface DownloadContextValue extends DownloadState {
  startDownload: (name: string) => void;
  markComplete: (name: string, trackId?: number) => void;
  markFailed: (name: string) => void;
  setPlaylistPreview: (p: PlaylistPayload | null) => void;
  setCreatedPlaylistId: (id: number | null) => void;
  setGeneratingPlaylist: (v: boolean) => void;
  setCreatingPlaylist: (v: boolean) => void;
  updateTrackStatus: (key: string, status: TrackStatus) => void;
  resetPlaylistState: () => void;
}

const DownloadContext = createContext<DownloadContextValue | null>(null);

export function DownloadProvider({ children }: { children: ReactNode }) {
  const [downloadingIds, setDownloadingIds] = useState<Set<string>>(new Set());
  const [completedIds, setCompletedIds] = useState<Set<string>>(new Set());
  const [completedTrackIds, setCompletedTrackIds] = useState<Record<string, number>>({});
  const [playlistPreview, setPlaylistPreview] = useState<PlaylistPayload | null>(null);
  const [playlistTrackStatus, setPlaylistTrackStatus] = useState<Record<string, TrackStatus>>({});
  const [createdPlaylistId, setCreatedPlaylistId] = useState<number | null>(null);
  const [generatingPlaylist, setGeneratingPlaylist] = useState(false);
  const [creatingPlaylist, setCreatingPlaylist] = useState(false);

  const startDownload = useCallback((name: string) => {
    setDownloadingIds((prev) => new Set(prev).add(name));
  }, []);

  const markComplete = useCallback((name: string, trackId?: number) => {
    setDownloadingIds((prev) => {
      const next = new Set(prev);
      next.delete(name);
      return next;
    });
    setCompletedIds((prev) => new Set(prev).add(name));
    if (trackId) {
      setCompletedTrackIds((prev) => ({ ...prev, [name]: trackId }));
    }
  }, []);

  const markFailed = useCallback((name: string) => {
    setDownloadingIds((prev) => {
      const next = new Set(prev);
      next.delete(name);
      return next;
    });
  }, []);

  const updateTrackStatus = useCallback((key: string, status: TrackStatus) => {
    setPlaylistTrackStatus((prev) => ({ ...prev, [key]: status }));
  }, []);

  const resetPlaylistState = useCallback(() => {
    setPlaylistPreview(null);
    setPlaylistTrackStatus({});
    setCreatedPlaylistId(null);
  }, []);

  return (
    <DownloadContext.Provider
      value={{
        downloadingIds,
        completedIds,
        completedTrackIds,
        playlistPreview,
        playlistTrackStatus,
        createdPlaylistId,
        generatingPlaylist,
        creatingPlaylist,
        startDownload,
        markComplete,
        markFailed,
        setPlaylistPreview,
        setCreatedPlaylistId,
        setGeneratingPlaylist,
        setCreatingPlaylist,
        updateTrackStatus,
        resetPlaylistState,
      }}
    >
      {children}
    </DownloadContext.Provider>
  );
}

export function useDownload() {
  const ctx = useContext(DownloadContext);
  if (!ctx) throw new Error("useDownload must be used within DownloadProvider");
  return ctx;
}
```

### A.6 Potential Pitfalls (Issue #13)

1. **Set comparison in render:** `Set.has()` still works correctly because the context provides the same Set instance reference. However, if a re-render happens before the Set is updated, the UI might show stale state. This is the same behavior as before — the context just doesn't make it worse.
2. **Polling intervals vs context:** The `pollRef.current` intervals are still managed inside RecsPage's `useEffect` cleanup. When navigating away, the cleanup function clears all intervals. This is correct — we don't want intervals running on pages that aren't visible. When the user returns, the intervals are gone but the `downloadingIds` Set still shows which items were "in progress." The user can re-trigger downloads if needed. This is acceptable behavior.
3. **Set serialization edge case:** React's `useState` with `Set` works fine for re-renders because `new Set(prev).add(x)` creates a new Set reference. The context pattern preserves this.

## B. Issue #14 — Track Struct Duplication

### B.1 Design Decision

**Chosen approach: Extract to shared `models` package**

Create `backend/internal/models/models.go` containing the `Track` struct, `TrackCols` constant, and `ScanTrack` function. Both `library` and `playlists` packages import from `models`.

**Alternatives considered and rejected:**

| Alternative | Why rejected |
|-------------|-------------|
| Playlists imports Track from library | Creates a dependency from playlists→library. Library is a "heavy" package with API struct, routes, DB queries. Playlists shouldn't depend on library's implementation details. Potential circular dependency if library ever needs playlist types. |
| Library imports Track from playlists | Architecturally backwards — library is the canonical source of track data. Worse than the previous option. |
| Leave duplicated (status quo) | Will definitely cause bugs when fields are added/changed in one but not the other. |

**Why models package wins:**
- Standard Go project layout pattern
- Zero dependencies — can be imported by any package
- No circular dependency risk
- Clean semantic boundary: models are data, packages are behavior

### B.2 Architecture

```
backend/internal/models/models.go     ← NEW: shared Track type
backend/internal/library/library.go   ← import models, delete local Track
backend/internal/playlists/playlists.go ← import models, delete local Track
```

The import path is `github.com/kevin/lexicon/internal/models` (matches the module path in go.mod).

### B.3 What Gets Extracted

```go
package models

// Track represents an audio track in the library.
// This is the canonical definition — library and playlists packages
// import it rather than defining their own copies.
type Track struct {
	ID          int64  `json:"id"`
	Title       string `json:"title"`
	Artist      string `json:"artist"`
	AlbumArtist string `json:"album_artist"`
	Album       string `json:"album"`
	TrackNo     int    `json:"track_no"`
	DiscNo      int    `json:"disc_no"`
	Year        int    `json:"year"`
	Genre       string `json:"genre"`
	DurationSec int    `json:"duration_sec"`
	MediaKind   string `json:"media_kind"`
	Mime        string `json:"mime"`
	SpotifyID   string `json:"spotify_id,omitempty"`
	ExternalURL string `json:"external_url,omitempty"`
}

// TrackCols is the SQL column list for scanning a Track row.
// Must match the column order in the ScanTrack function.
const TrackCols = `id,title,IFNULL(artist,''),IFNULL(album_artist,''),IFNULL(album,''),IFNULL(track_no,0),IFNULL(disc_no,0),IFNULL(year,0),IFNULL(genre,''),IFNULL(duration_sec,0),media_kind,IFNULL(mime,''),IFNULL(spotify_id,''),IFNULL(external_url,'')`

// RowScanner is satisfied by *sql.Row and *sql.Rows.
type RowScanner interface {
	Scan(...interface{}) error
}

// ScanTrack scans a Track from a database row.
func ScanTrack(rows RowScanner) (Track, error) {
	var t Track
	err := rows.Scan(&t.ID, &t.Title, &t.Artist, &t.AlbumArtist, &t.Album, &t.TrackNo, &t.DiscNo, &t.Year, &t.Genre, &t.DurationSec, &t.MediaKind, &t.Mime, &t.SpotifyID, &t.ExternalURL)
	return t, err
}
```

## C. Issue #15 — MIME Type Validation After Download

### C.1 Design Decision

**Chosen approach: ffprobe-based validation with file-extension fallback**

After a successful download (in both `run()` and `runSearch()`), scan the output directory for recently modified files, run `ffprobe` to check the actual format, and log warnings on mismatch. If ffprobe isn't available, fall back to checking the file extension.

**Alternatives considered and rejected:**

| Alternative | Why rejected |
|-------------|-------------|
| http.DetectContentType | Designed for web content detection. Audio format detection is limited — returns "application/octet-stream" for many audio formats. |
| Parse yt-dlp output for destination filename | Fragile string parsing, output format varies across yt-dlp versions. |
| Validate at scan time (in scanner) | Scanner doesn't know the "expected" format. Validating at download time catches the problem closer to the source. |
| Just check file extension | Extension ≠ format. A .mp3 file could contain AAC data. |

**Why ffprobe + fallback wins:**
- ffprobe is already bundled with the app (part of ffmpeg tools)
- Most reliable method — reads actual codec/container data
- Graceful degradation when ffprobe unavailable
- Non-blocking — warns in the job log, doesn't fail the download
- Matches the existing pattern of calling external tools

```go
// validateOutputFormat checks recently-downloaded files in the output directory
// to verify their actual format matches the expected format. Uses ffprobe when
// available, falls back to file extension check. Non-blocking — only logs warnings.
func (a *API) validateOutputFormat(job *Job, expectedFormat string) {
	outputDir := strings.ReplaceAll(a.cfg.Output, "\\", "/")
	if outputDir == "" {
		return
	}

	// Find files modified in the last 30 seconds (covers the download window)
	cutoff := time.Now().Add(-30 * time.Second)
	entries, err := os.ReadDir(outputDir)
	if err != nil {
		a.appendLog(job, fmt.Sprintf("[validate] cannot read output dir: %v", err))
		return
	}

	ffprobePath := ""
	if a.cfg.FfmpegBin != "" {
		// ffprobe is typically in the same directory as ffmpeg
		dir := filepath.Dir(a.cfg.FfmpegBin)
		candidate := filepath.Join(dir, "ffprobe.exe")
		if _, err := os.Stat(candidate); err == nil {
			ffprobePath = candidate
		} else {
			// Try without .exe extension (Linux/WSL)
			candidate = filepath.Join(dir, "ffprobe")
			if _, err := os.Stat(candidate); err == nil {
				ffprobePath = candidate
			}
		}
	}

	for _, entry := range entries {
		if entry.IsDir() {
			continue
		}
		info, err := entry.Info()
		if err != nil {
			continue
		}
		if info.ModTime().Before(cutoff) {
			continue
		}

		filePath := filepath.Join(outputDir, entry.Name())
		ext := strings.TrimPrefix(strings.ToLower(filepath.Ext(entry.Name())), ".")

		var actualFormat string
		var method string

		if ffprobePath != "" {
			// Primary: use ffprobe
			ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
			cmd := exec.CommandContext(ctx, ffprobePath,
				"-v", "quiet",
				"-show_entries", "format=format_name",
				"-of", "default=noprint_wrappers=1:nokey=1",
				filePath,
			)
			out, err := cmd.Output()
			cancel()
			if err == nil {
				actualFormat = strings.TrimSpace(string(out))
				// ffprobe may return multiple formats (e.g. "mp3,mpa").
				// Take the first one.
				if comma := strings.Index(actualFormat, ","); comma >= 0 {
					actualFormat = actualFormat[:comma]
				}
				actualFormat = strings.ToLower(actualFormat)
				method = "ffprobe"
			}
		}

		if actualFormat == "" {
			// Fallback: check file extension
			actualFormat = ext
			method = "extension"
		}

		// Normalize formats for comparison
		expected := strings.ToLower(strings.TrimSpace(expectedFormat))
		// Handle common aliases
		normalizeFormat := func(f string) string {
			switch f {
			case "mp3", "mpa", "mpeg", "mpga":
				return "mp3"
			case "aac":
				return "aac"
			case "opus":
				return "opus"
			case "flac":
				return "flac"
			case "ogg", "vorbis":
				return "ogg"
			case "wav":
				return "wav"
			case "m4a", "mp4":
				return "m4a"
			default:
				return f
			}
		}

		normExpected := normalizeFormat(expected)
		normActual := normalizeFormat(actualFormat)

		if normExpected != normActual {
			a.appendLog(job, fmt.Sprintf(
				"[validate] WARNING: format mismatch for %s — expected %s, got %s (detected via %s)",
				entry.Name(), expectedFormat, actualFormat, method,
			))
		} else {
			a.appendLog(job, fmt.Sprintf(
				"[validate] format verified: %s (%s)",
				entry.Name(), actualFormat,
			))
		}
	}
}
```

### C.6 Potential Pitfalls (Issue #15)

1. **Multiple concurrent downloads** — the 30-second window could pick up files from another concurrent download. Mitigation: this is a non-blocking warning, not a failure. The validation is best-effort. In practice, Lexicon downloads are sequential (one at a time).
2. **ffprobe path on Windows** — `ffprobe.exe` must be in the same directory as `ffmpeg.exe`. The code derives the ffprobe path from the ffmpeg path. If ffmpeg is on PATH rather than an absolute path, this won't work. Mitigation: also try bare `"ffprobe"` and let exec.LookPath find it. (We rely on os.Stat which needs full path — for PATH-based lookup, we'd need `exec.LookPath("ffprobe")`.)
3. **ffprobe format aliases** — ffprobe reports `"mp3"` but yt-dlp's `--audio-format mp3` might produce a file ffprobe reports as `"mp3,mpa"`. The code takes the first comma-separated value and normalizes it. This should handle all common cases.
4. **Subprocess timeout** — ffprobe has a 5-second timeout per file. For a large directory with many recent files, this could add up. In practice, the output directory has a handful of files and ffprobe on a single audio file takes <100ms.
